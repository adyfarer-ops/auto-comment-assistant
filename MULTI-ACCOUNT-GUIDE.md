# 多账号评论系统使用说明

## 端口映射规则

| 序号 | 本地端口 | SSH 端口 | 用户数据目录 |
|------|----------|----------|--------------|
| 1 | 9001 | 62001 | account_1 |
| 2 | 9002 | 62002 | account_2 |
| 3 | 9003 | 62003 | account_3 |
| ... | ... | ... | ... |
| 50 | 9050 | 62050 | account_50 |

## 使用流程

### 1. 本地启动 Chrome（Windows）

```batch
start-chrome-by-index.bat 1 9001 62001 account_1
```

参数说明：
- `%1` - 序号（1-50）
- `%2` - 本地端口（9001-9050）
- `%3` - SSH 端口（62001-62050）
- `%4` - 用户数据目录名

### 2. 本地启动 Chrome（Mac/Linux）

```bash
./start-chrome-by-index.sh 1 9001 62001 account_1
```

### 3. 飞书表格填写

在飞书表格中填写：
- **序号**：1-50 之间的数字
- **产品名称**：产品名称
- **产品链接**：抖音/小红书等链接
- **评论话术**：要发送的评论内容
- **点击"去评论"按钮**

### 4. 系统自动执行

1. 服务器根据序号确定 SSH 端口（62001-62050）
2. 连接本地 Chrome（通过 SSH 隧道）
3. 打开产品链接
4. 执行评论
5. 截图保存

## 首次使用设置

对于每个序号（如 1-10），需要：

1. **飞书表格填写序号** `1`
2. **本地启动对应 Chrome**：
   ```batch
   start-chrome-by-index.bat 1 9001 62001 account_1
   ```
3. **手动登录**抖音账号
4. **关闭 Chrome**（登录态已保存到 `C:\chrome_profiles\account_1`）
5. **下次使用序号 1** 时自动恢复登录态

## 注意事项

1. **每个序号对应一个独立的 Chrome 用户数据目录**
2. **首次使用必须手动登录**，之后自动恢复登录态
3. **SSH 隧道必须保持连接**，服务器才能访问本地 Chrome
4. **端口范围**：
   - 本地端口：9001-9050
   - SSH 端口：62001-62050

## 故障排查

### 连接失败
- 检查本地 Chrome 是否已启动
- 检查 SSH 隧道是否已建立
- 检查端口是否正确（序号对应）

### 登录态丢失
- 检查用户数据目录是否存在
- 重新手动登录一次

### 端口冲突
- 确保每个序号使用不同的端口
- 检查是否有其他程序占用了端口
